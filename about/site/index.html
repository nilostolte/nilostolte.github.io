About this site

This is an experimental site containing only vector information. The contents are near WYSIWYG with text and images zoomable. An extra mile has been taken to make it mobile ready, but HTML 5 Canvas images are not yet zoomable on mobiles, only on desktops when browser window size changes (for example, putting the browser window in full screen). This feature was implemented in Javascript.

The goal of this site is to demonstrate that modern web sites can be all vectorial only using standard HTML, CSS and Javascript. Constructing sites in this way creates desirable features, especially on mobiles, since it allows its contents to be seen and read in larger sizes without any change in the formatting. It's easier for visual impaired persons to be able to access the information more confortably.

On the other hand, vector web sites are also interesting to apply the GUI concepts that have been praised in this site. Although, coding for the web is different, involving complex CSS formatting and HTML structuring, as well as a script language used as a programming language, the same ideas are also valid here. The main problem is that web programming became an universe in itself, completely isolated from the rest of the computing world. One could imagine the use of all this know-how on a desktop, using webkit for example. This solution would be too cumbersome, though, and limited to a certain programming language as well as to desktops only.

The simpler solution is to do the opposite, that is, to code the application in a universal programming language and generating code to the web directly from that programming language. This is the idea behind WebAssembly. That's where vector graphics GUIs seem to adapt better to create portable and universal GUIs with a single code.

CSS code

CSS coding was prefered for formatting this site, although some old HTML structures (such as tables) were sometimes used instead for confort. The reason for using CSS rather than HTML to do the same thing is that CSS is more flexible and much nearer to what vector graphics is about than HTML.
